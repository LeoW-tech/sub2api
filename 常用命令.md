# 常用命令

## 使用约定

当前 Linux 正式环境的仓库根目录：

```bash
cd /srv/sub2api/repo
```

当前 Linux 正式运行时目录：

```bash
/srv/sub2api/runtime/stable
```

当前正式环境的对外与本地入口：

- 本机访问：`http://127.0.0.1:8080/`
- 局域网访问：`http://192.168.31.214:8080/`
- 公网访问：`https://api.cloudalpha.top/`

## 最常用：一键重启 / 一键重建重启 stable

```bash
# 一键重启正式 stable
sudo systemctl restart sub2api-stable

# 一键重建并重启正式 stable
sudo /usr/local/bin/sub2api-stable-rebuild
```

说明：

- `sub2api-stable` 是 Linux 正式环境的 `systemd` 服务
- `restart` 会按正式 compose 配置重启运行面
- `sub2api-stable-rebuild` 会重建 `sub2api` 镜像并自动重启正式运行面
- 这两条命令都是 Linux 正式环境入口，不再依赖 macOS 本地脚本

## 正式环境：查看状态

```bash
sudo systemctl status sub2api-stable --no-pager -l
sudo systemctl status sub2api-door-gateway --no-pager -l
sudo systemctl status cloudflared --no-pager -l

sudo /usr/local/bin/sub2api-stable-compose ps
sudo docker ps --format '{{.Names}}\t{{.Image}}\t{{.Status}}\t{{.Ports}}'
```

## 正式环境：查看日志

```bash
sudo /usr/local/bin/sub2api-stable-compose logs -f --tail=200 sub2api postgres redis
sudo journalctl -u sub2api-stable -n 200 --no-pager -l
sudo journalctl -u sub2api-door-gateway -n 200 --no-pager -l
sudo journalctl -u cloudflared -n 200 --no-pager -l
```

## 正式环境：健康检查

```bash

curl http://192.168.31.214:8080/health
curl https://api.cloudalpha.top/health
curl http://127.0.0.1:19080/health
curl http://127.0.0.1:19080/export/sub2api
```

## 正式环境：Compose 维护命令

```bash
# 查看 compose 运行面
sudo /usr/local/bin/sub2api-stable-compose ps

# 启动 / 拉起正式运行面
sudo /usr/local/bin/sub2api-stable-compose up -d

# 停止正式运行面
sudo /usr/local/bin/sub2api-stable-compose down

# 重建 sub2api 镜像
sudo /usr/local/bin/sub2api-stable-compose build sub2api

# 只重启 sub2api 容器
sudo /usr/local/bin/sub2api-stable-compose up -d sub2api
```

## 正式环境：door-gateway

```bash
sudo systemctl restart sub2api-door-gateway
sudo systemctl status sub2api-door-gateway --no-pager -l
curl http://127.0.0.1:19080/health
curl http://127.0.0.1:19080/export/sub2api
```

## 正式环境：开机自启 / 守护

```bash
sudo systemctl is-enabled sub2api-stable
sudo systemctl is-enabled sub2api-door-gateway
sudo systemctl is-enabled cloudflared

sudo docker inspect sub2api_stable-sub2api-1 --format '{{.HostConfig.RestartPolicy.Name}}'
sudo docker inspect sub2api_stable-postgres-1 --format '{{.HostConfig.RestartPolicy.Name}}'
sudo docker inspect sub2api_stable-redis-1 --format '{{.HostConfig.RestartPolicy.Name}}'
```

说明：

- `sub2api-stable.service` 负责系统启动后自动拉起正式运行面
- `sub2api-door-gateway.service` 负责系统启动后自动拉起门控
- Docker 容器本身使用 `restart: unless-stopped`
- 因此系统重启、Docker 守护重启、容器异常退出这几类场景，都会自动恢复
